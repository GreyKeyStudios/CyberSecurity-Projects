# SOC Playbooks

This directory contains reusable security playbooks for common incident types. These playbooks provide step-by-step procedures for SOC analysts to follow when responding to security incidents.

## Available Playbooks

### Phishing Response Playbook
**File:** `Phishing-Response-Playbook.md`  
**Use Case:** Responding to phishing email reports and investigations

### Brute Force Response Playbook
**File:** `Brute-Force-Response-Playbook.md`  
**Use Case:** Responding to brute force login attempts

### Malware Response Playbook
**File:** `Malware-Response-Playbook.md`  
**Use Case:** Responding to malware detections on endpoints

### Account Compromise Playbook
**File:** `Account-Compromise-Playbook.md`  
**Use Case:** Responding to suspected account compromise

## How to Use Playbooks

1. **Identify the incident type** from the alert
2. **Locate the appropriate playbook** in this directory
3. **Follow the steps** in order
4. **Document actions taken** in your ticket/incident report
5. **Escalate** if the playbook indicates escalation criteria

## Playbook Structure

Each playbook follows this structure:
- **Purpose:** What this playbook covers
- **Prerequisites:** Required access/tools
- **Triage Steps:** Initial assessment
- **Investigation Steps:** Evidence gathering
- **Containment Steps:** Stopping the threat
- **Remediation Steps:** Cleaning up and securing
- **Escalation Criteria:** When to escalate
- **References:** Related resources

---

> **Note:** These playbooks are templates based on industry best practices. Adapt them to your organization's specific tools, processes, and policies.
